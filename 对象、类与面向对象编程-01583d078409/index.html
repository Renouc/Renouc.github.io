对象、类与面向对象编程

对象就是一组没有特点循序的值。对象的每个属性或方法都由一个名称来标识，这个名称映射到一个值。可以把 ECMAScript 对象想象为一张散列表，其中的内容就是一组 名/值 对，值可以是数据或函数。

理解对象

创建自定义对象的方法

  • 创建一个 Object 实例，然后再为其添加属性和方法
  • 使用字面量创建对象

属性的类型

有一些内部属性来描述对象属性的特性。为了将某个特性标识为内部特性，规范会用两个中括号把特性名称括起来。

数据属性

数据属性包含一个保存数据的位置。值会从这个位置读取，也会写入到这个位置。数据属性有四个特性描述它们的行为：

  • [[Configurable]]：表示能否通过 delete 删除属性，能否修改属性的特性，或者能否把属性修改为访问器属性。默认为 true。

  • [[Enumerable]]：表示能否通过 for-in 循环返回属性。默认为 true。默认为 true。

  • [[Writable]]：表示能否修改属性的值。默认为 true。

  • [[Value]]：包含这个属性的数据值。读取属性时，从这个位置读取数据值。写入属性时，把新值保存在这个位置。默认为 undefined。

在调用 Object.defineProperty() 时，configurable、enumerable 和 writable 的值如果不指定，则都默认为 false

访问器属性

访问器属性不包含数据值。访问器属性有四个特性描述它们的行为。

  • [[Configurable]]：表示能否通过 delete 删除属性，能否修改属性的特性，或者能否把属性修改为数据属性。默认为 true。

  • [[Enumerable]]：表示能否通过 for-in 循环返回属性。默认为 true。

  • [[Get]]：在读取属性时调用的函数。默认为 undefined。

  • [[Set]]：在写入属性时调用的函数。默认为 undefined。

定义多个属性

Object.defineProperties() 方法可以一次定义多个属性。

读取属性的特性

Object.getOwnPropertyDescriptor() 方法可以读取一个属性的特性。

Object.getOwnPropertyDescriptors() 方法可以读取对象的所有属性的特性。

合并对象

Object.assign() 方法用于对象的合并，将源对象（source）的所有可枚举属性，复制到目标对象（target）。

创建对象

工厂模式

工厂模式（Factory Pattern）是创建型设计模式之一，它允许我们使用一个函数来创建对象。

构造函数模式

像 Object 和 Array 这样的原生构造函数，运行时直接使用。也可以自定义构造函数

要创建 Person 实例，应使用 new 操作符。以这种方式调用构造函数会执行如下操作：

  1. 在内存中创建一个新对象
  2. 这个新对象内部的[[Prototype]]特性被赋值为构造函数的 prototype 属性
  3. 构造函数内部的 this 被赋值为这个新对象（即 this 指向新对象）
  4. 执行构造函数内部的代码（给对象添加属性）
  5. 如果构造函数返回非空对象，则返回该对象；否则返回刚创建的新对象

创建出的实例对象上都有 constructor 属性，指向构造函数。

如果构造函数不需要传递参数，也可以省略括号

原型模式

所有函数上都有 prototype 属性，指向函数的原型对象。

如果使用 new 操作符创建实例对象，则创建出的实例对象共享原型对象上的属性和方法。
