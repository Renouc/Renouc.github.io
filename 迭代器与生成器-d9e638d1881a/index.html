迭代器与生成器

迭代器和生成器是 JavaScript 中两个重要的概念，它们允许我们遍历数据结构，并执行一些操作。

迭代器

可迭代协议

实现 Iterable 接口的对象，可以迭代。即该对象实现了 Symbol.iterator 方法。

很多数据结构都实现了这个接口，如 字符串、数组、映射、集合、arguments 对象、NodeList 对象等。

Symbol.iterator 是一个工厂函数，返回一个迭代器。

实际写代码的过程中，不需要显式调用这个工厂函数来生成迭代器。实现可迭代协议的所有类型都会自动兼容接收可迭代对象的任何语言特性。接收可迭代对象的原生语言特性包括：

  • for of 循环
  • 数组结构
  • 扩展运算符
  • Array.from()
  • 创建集合
  • 创建映射
  • Promise.all() 接收由 Promise 组成的可迭代对象
  • Promise.race() 接收由 Promise 组成的可迭代对象
  • yield* 操作符

迭代器协议

Symbol.iterator 方法返回一个迭代器对象。

该迭代器对象包含一个 next 方法，该方法返回一个 IteratorResult 对象，该对象包含 value 和 done 属性。

这里通过创建迭代器并调用 next 方法来获取迭代器的下一个值，直至不再产生新值。只要迭代器到达 done: true 状态，后续调用 next() 就一直返回同样的结果了。

注意：迭代器维护着一个指向可迭代对象的引用，因此迭代器会阻止垃圾回收程序回收可迭代对象。

自定义迭代器

自定义迭代器需要实现 Symbol.iterator 方法，该方法返回一个迭代器对象。

原生支持的可迭代对象，调用Symbol.iterator方法，返回一个迭代器对象。这个迭代器对象也实现了Symbol.iterator方法，返回自身。

例如：数组

提前终止迭代器

迭代器对象中有一个方法 return，在迭代器关闭时执行该方法。类似回调函数，在迭代器关闭时被调用。

执行迭代的结构在想让迭代器知道它不想遍历到可迭代对象耗尽时，就可以 “关闭” 迭代器。可能的情况包括：

  • for-of 循环通过 break、continue、return 或 throw 提前退出
  • 解构操作并未消费到所有可迭代对象中的元素

return 方法必须返回一个有效的 IteratorResult 对象。简单情况下，只返回 {done: true}即可。

如果迭代器没有关闭，则可以继续从上次离开的位置继续迭代。例如：数组的迭代器就是不能关闭的

仅仅给一个不可关闭的迭代器增加 return 方法 并不能让它变成一个可关闭的迭代器。这是因为调用 return 方法不会强制迭代器进入关闭状态。即便如此，return 方法 还是会被调用。

生成器

生成器是 ES6 新增的一个特性，拥有在一个函数块内暂停和恢复代码执行的能力。

生成器的形式是一个函数，函数名称前面加一个星号（*）表示它是一个生成器。只要是可以定义函数的地方，就可以定义生成器。

  • 生成器函数声明
  • 生成器函数表达式
  • 作为对象字面量方法的生成器函数
  • 作为类的实例方法的生成器函数
  • 作为类的静态方法的生成器函数

注意：箭头函数不能用来定义生成器函数。

调用生成器函数会返回一个生成器对象。生成器对象也实现了 Iterator 接口，因此可以作为迭代器使用。

生成器对象的默认迭代器是自引用的
